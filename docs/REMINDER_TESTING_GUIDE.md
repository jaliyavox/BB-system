**PAYMENT REMINDER SYSTEM - IMPLEMENTATION GUIDE**

## What Was Implemented

### 1. **Reminder Service** (`backend/src/payment/services/reminderService.js`)
- `createPrePaymentReminder()` - Creates reminder 5 days before payment cycle starts
- `createOverdueReminder()` - Creates reminder after payment window opens
- `generateAllReminders()` - Loops through all active payment cycles and creates appropriate reminders
- `createTestReminder()` - Manual test function to create reminders instantly (no waiting)
- `cleanupOldReminders()` - Removes reminders older than 30 days

### 2. **Database Model Updates** (`backend/src/models/Notification.js`)
Added new fields:
- `reminderType`: pre_payment | overdue (for categorizing reminders)
- `autoGenerated`: boolean (to distinguish auto-generated from manual notifications)
- `paymentCycleId`: Reference to PaymentCycle
- `cycleNumber`: Payment cycle number
Updated enum types to include:
- `payment_pre_payment`
- `payment_overdue_reminder`

### 3. **Test Endpoints** (New Routes)
- **POST** `/api/notifications/test-reminder`
  - Body: `{ "type": "pre_payment" }` or `{ "type": "overdue" }`
  - Creates instant test reminder for authenticated user
  
- **POST** `/api/notifications/generate-all`
  - Generates reminders for ALL students based on payment cycle dates
  - Pre-payment: Exactly 5 days before start date
  - Overdue: After start date and before due date
  
- **POST** `/api/notifications/cleanup-old`
  - Removes auto-generated reminders older than 30 days

### 4. **Automatic Scheduler** (`backend/src/utils/reminderScheduler.js`)
- Runs daily at server startup
- Automatically checks all payment cycles and creates reminders
- Includes cleanup routine

### 5. **Server Integration** (`backend/src/server.js`)
- Reminder scheduler starts when server starts
- Scheduler stops when server shuts down

## HOW AUTOMATIC REMINDERS WORK

### When Server Starts:
1. `reminderScheduler.js` initializes
2. Runs reminder generation IMMEDIATELY
3. Checks all active payment cycles
4. Creates reminders based on dates:
   - TODAY == (START_DATE - 5 DAYS) → Create pre-payment reminder
   - TODAY > START_DATE AND TODAY < DUE_DATE → Create overdue reminder

### Every 24 Hours:
- Scheduler runs again
- Creates any new reminders needed
- Randomly runs cleanup (removes old reminders)

### Current Date: March 31, 2026
- Payment cycle starts April 29
- 5 days before = **April 24**
- Pre-payment reminder will auto-create on April 24
- Overdue reminder will auto-create on April 29

## TROUBLESHOOTING

**Q: Test endpoint returns 500 error**
- Check console for "No active payment cycle found"
- Solution: Ensure user has an active payment cycle in database

**Q: Reminders not showing in frontend**
- Verify API endpoint is working with GET /api/notifications
- Check browser console for network errors
- Refresh the page after creating reminder

**Q: "Reminder already exists" error**
- Each user can have only ONE pre-payment and ONE overdue reminder per cycle
- Delete the old one first: DELETE /api/notifications/{notificationId}

**Q: Database error with paymentCycleId**
- Ensure Notification model is updated with new fields
- Restart backend after model changes
